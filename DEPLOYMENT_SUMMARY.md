### Getting Exact Values

#### PostgreSQL Credentials
1. Go to Render Dashboard → PostgreSQL → xeno-postgres
2. Copy "Internal Database URL"
3. Extract values from URL: `postgresql://postgres:PASSWORD@HOST:5432/xeno`
   - `DB_PASSWORD` = PASSWORD
   - `DB_HOST` = HOST

#### Redis Credentials
1. Go to Render Dashboard → Redis → xeno-redis
2. Copy "Redis URL"
3. Extract values from URL: `redis://:PASSWORD@HOST:6379`
   - `REDIS_URL` = full URL or construct with password
   - Format: `redis://:<password>@<host>:6379/0`

#### Groq API Key
1. Go to [Groq Console](https://console.groq.com)
2. Create API key
3. Copy key (starts with `gsk_`)

## Exact Redis Connection Settings

### Connection String Format
```python
# Format
redis://:<password>@<host>:6379/0

# Example
redis://:red-xxxxx@red-xxxxx.oregon.render.com:6379/0
```

### Queue Configuration
```python
# Queue Name
"default"

# Retry Policy (in code)
Retry(max=3, interval=[60, 120, 240])

# Worker Connection
from redis import Redis
from rq import Queue, Worker

redis_conn = Redis.from_url(settings.REDIS_URL)
queue = Queue("default", connection=redis_conn)
worker = Worker([queue], connection=redis_conn)
```

### Connection Resilience
- **Auto-reconnect**: Redis-py has built-in reconnection logic
- **Retry policy**: 3 retries with exponential backoff
- **Connection timeout**: Handled by Redis client defaults
- **Health checks**: Monitor Redis status in Render dashboard

## Remaining Deployment Blockers

### 1. Alembic Hardcoded Database URL
**Issue**: `backend/alembic.ini` contains hardcoded database URL
**Location**: Line 89 in `alembic.ini`
**Current**: `sqlalchemy.url = postgresql://postgres:Dev%4011976@db:5432/xeno`
**Required Fix**: This is overridden by `alembic/env.py` which uses environment variables, but the file should be cleaned up
**Action**: Update `alembic.ini` to use placeholder or remove hardcoded URL
**Priority**: Medium (env.py handles this correctly)

### 2. Frontend Default API URL
**Issue**: `xeno-data-hub/next.config.ts` has default `localhost:8000`
**Location**: Line 4 in `next.config.ts`
**Current**: `const apiOrigin = process.env.NEXT_PUBLIC_API_URL ?? "http://localhost:8000";`
**Required Fix**: Already handled by environment variable in production
**Priority**: Low (environment variable overrides this)

### 3. Backend Default Debug Mode
**Issue**: Default DEBUG mode is `true` in settings
**Location**: `backend/app/config/settings.py` line 11
**Current**: `debug: bool = field(default_factory=lambda: os.getenv("DEBUG", "true").lower() == "true")`
**Required Fix**: Change default to `false` for production safety
**Priority**: Medium (environment variable overrides this)

### 4. Missing Production Error Handling
**Issue**: Some error responses may expose stack traces in debug mode
**Required Fix**: Ensure debug mode is false in production
**Priority**: Medium (handled by DEBUG=false env var)

### 5. File Storage Ephemeral Nature
**Issue**: Render and Railway have ephemeral file systems
**Impact**: Uploaded files and outputs may be lost on redeployment
**Required Fix**: Consider S3 or similar for persistent storage in production
**Priority**: Low (acceptable for MVP, upgrade later)

### 6. No Health Check for Worker
**Issue**: Railway worker has no HTTP health check endpoint
**Impact**: Harder to monitor worker status
**Required Fix**: Add health check endpoint to worker or use Railway's process monitoring
**Priority**: Low (Railway monitors process health)

## Recommended Post-Deployment Actions

### Immediate (After First Deployment)
1. **Test complete workflow**: Upload → Process → Download
2. **Verify worker connectivity**: Check Railway logs for Redis connection
3. **Test error handling**: Upload invalid file to check error reporting
4. **Monitor resource usage**: Check Render/Railway dashboards
5. **Set up alerts**: Configure error notifications

### Short Term (First Week)
1. **Monitor logs regularly**: Check for any unexpected errors
2. **Performance testing**: Test with larger files
3. **Database optimization**: Review query performance
4. **Security audit**: Review all environment variables and access
5. **Backup verification**: Ensure database backups are working

### Long Term (Production Scale)
1. **Implement persistent storage**: Move file storage to S3
2. **Add monitoring**: Integrate with monitoring service (Sentry, DataDog)
3. **CDN integration**: Use CDN for static assets
4. **Load testing**: Test with concurrent users
5. **Disaster recovery**: Test backup and restore procedures
